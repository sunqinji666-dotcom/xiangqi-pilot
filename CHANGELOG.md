# XiangqiPilot 变更日志

## 版本规则

- 主版本.次版本.修订号（例如 `v0.5.0`）
- macOS 应用中的短版本号与 Git 标签保持一致
- `CFBundleVersion` 使用递增构建号
- 回滚可以使用对应标签，例如：`git checkout v0.5.0`

---

## v0.5.0 — 本机 AI 协同、连接配置与识别安全加固

**日期：** 2026-07-21
**macOS 构建号：** 5

### 新增

- 新增只监听本机 Unix Socket 的 AI 协同桥，AI 可提交 FEN，驾驶舱负责规则校验和引擎候选分析
- 新增连接配置持久化、棋盘变化检测、局面连续性和位置同步组件
- 新增宽立棋盘方向检测、客户端能力适配和任意局面可信策略
- 新增专用签名钥匙串流程，减少重新构建后丢失 macOS 权限的问题
- 新增 Pikafish 深度、选择深度、绝杀、节点数、NPS 和耗时解析
- 新增返回首页、重新选择目标和协同状态展示

### 安全调整

- 标准新局优先使用 32 个占位交点建立规则局面，后续只接受唯一合法走法对应的严格变化
- 未安装经过验证的本地 14 类棋子模型时，任意中残局只能生成待确认草稿，不能静默成为可信基线
- 手工修正棋子时检查棋子库存上限，避免凭空增加不可能的棋子
- AI 协同模式不申请读屏和辅助功能权限，也不执行远程点击指令
- 自动点击继续绑定目标窗口、局面、画面序号和当前合法候选

### 验证

- 109 个 Swift 测试全部通过
- 2 个 Node.js 桥接脚本测试全部通过
- 敏感信息扫描未发现真实 API Key、私钥或密码

## v0.3.0 — 象棋巫师连续对局同步与安全校验

**日期：** 2026-07-20
**macOS 构建号：** 3

- 新增象棋巫师可访问性树与 Vision 双通道棋谱读取
- 支持根据最新棋谱行解析唯一合法走法，并识别胜负和棋终局提示

- 窗口平移时更新点击坐标，只有缩放时才要求重新校准
- 加强自动执行去重、状态门控和执行前复核
